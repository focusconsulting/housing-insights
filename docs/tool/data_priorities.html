Data Priorities for the Housing Insights Tool

There are many relevant data sources we could incorporate into the Housing Insights tool. We need the affordable housing community to help us decide which to focus on. Please take our one page survey to share what you would find most useful in a tool designed to help prioritize efforts in affordable housing preservation.

What can we do with this kind of data?

As one example of the type of analysis we envision this tool will be able to provide, we've already connected the Preservation Catalog to DC's tax assessment database. Although just an example, we hope that this type of analysis can make it easier to better prioritize the time and money spent on affordable housing preservation.

How much is DC's affordable housing worth?

Not all affordable housing units will cost the same to preserve if their building owners decide to sell. To get a sense of this difference, we connected the Preservation Catalog to DC's Real Property Tax Database to find out the assessed tax value of all the affordable buildings in DC.

This graph sums the tax value of all affordable developments in the Preservation Catalog by neighborhood cluster. The values on both the map and bar chart represent the average assessed value per affordable unit.

Note this is draft data - while close to accurate, we haven't completed full quality assurance to make sure all buildings are accurately represented. This should, however, give you a sense of the type of information the Housing Insights tool will provide.

Technical Notes

This is draft analysis, and should not be quoted or used outside of this page until a final analysis is published. The purpose of this page is to show Preservation Network Members the type of information that we intend to include in the final Housing Insights tool.

Data on this page comes from matching all 'Active' projects in the Preservation Catalog to their corresponding tax assessment records from the DC Government. A few known issues with the data include:

  • While considered mostly accurate, comprehensive data quality checks have not yet been performed.
  • Some projects are missing unit counts, which will make averages appear higher than they should be.
  • Some projects could not be found in the tax database, which will make totals and averages appear lower than they should be.
  • Tax values represent the entire project, while average tax value refers only to the affordable units. This will make average values higher than they should be for buildings with lots of unsubsidized units (e.g. LIHTC).
  • Tax assessments typically lag real estate market prices by a significant amount, so the total market value is likely higher than what is represented.
